I installed pyserial and used the miniterm module within it to test the 4030XL to find out why the spindle motor does not turn on when it receives and M3 (Start spindle clockwise). It turns out that one must explicitly initialize a speed using an Sxxxx command OR one must use an M3 Sxxxx. One other disconcerting implication of this is that if one issues an M3, and an Sxxxx has NOT been previous issued, and then one issues an Sxxxx, the spindle motor will turn on, so it's important to issue an M2 (or M5) before issuing an Sxxxx to make sure no accidental turning-on of the spindle motor occurs.

To run miniterm I used the following syntax: 
python -m serial.tools.miniterm -e /dev/ttyUSB0 115200
The -e stands for "echo" which prints what you type.
